What's new

      Emojis

      • Upload, edit and delete Emojis!
        • Its now possible to use emojis! You can go to the Server Settings » Media Settings » Emojis and upload, edit or delete emojis. You can insert emojis by using the new Emoji button next to the message box or by typing its name in between two : . Example: Emoji name is wave, so you'd type :wave:. It works like on discord.
        • Currently emojis inserted via the Emoji Box or written manually are not shown in the message box, but it will be shown in the chat!

      Bans

      • Ban + Banlist Feature
        • I've finally implemented the ban feature and a banlist. Banned Members can be found in Server Settings » Banned Members.
        • A new feature will be added in the future which will allow to ban IPs manually too.

      Notifications

      • Better Notifications
        • Proper Notification Boxes have been implemented. No (or less) ugly alert boxes! Whats a alert box? click here to find out! And this is the new box!

      Mentions

      • You can mention users now
        • You can mention users with the same format as in Discord using <@useridhere> or by right clicking the member list » Mention User

      Sorting Update

      • Sorting of Groups, Categories and Channels
        • Change Hierarchy/Sorting of..
          • ... Groups
          • ... Categories
          • ... Channels

      Markdown

      • Improved Markdown
        • Markdown works a lot better now. You can even post image links that dont include a file extention now and it will still be embedded. Also, embeds will automatically retry to change a link into a image. If its not a image or unkown it will be turned into a url
        • Fixed a bug where the markdown would split a link and puts a List Symbol in front of it, breaking the entire embed

      Member Kicks

      • Improved Kicking members
        • Its not possible to kick users that have a higher role than the person initiating the kick.

      Better Highlighting

      • Group Highlighting
        • In addition to the round border on selected groups, any group that is not selected will now be grayed out. Once selected the colors come back
      • Category/Channel Highlighting
        • The current category and channel is now brighter then all the other channels.

      Server Settings

      • Updated Setting Display
        • A user now only sees the settings he's allowed to modify. For example, if you create a Moderator Role and give its the permission Manage Channels, it can only see the Channel Sorting option in the server settings page

      Bug Fixes

      • Fixed some bugs and improved things

      Role Update

      • Role Editor
        • Edit Roles
        • Create Roles
        • Delete Roles
        • Change Role Color
        • Change Role Name
        • Change Role Display Setting (show in member list)
        • Change Role Permissions
        • Change Role hierarchy (role sorting)

      Channel Update

      • Edit Channel
        • Its now possible to edit a channel by right clicking it and selecting "Edit Channel". There you can change the channel name and the permissions for roles.

      Server Update

      • SSL Support
        • SSL is required if you host your server on a maschine and access it from another device. Otherwise some features like the "Copy User ID" will not work as browsers like Chrome will block it on "unsecure sources". Also SSL will encrypt the Traffic which is another big plus. A few new settings have been added to the config file.

      General Changes

      • Member List Update
        • The Member list will only show members that are allowed to view the current channel.
      • Channel List Update
        • If a category doesnt have any channel a member/role is allowed to see, the entire category is hidden from the channel tree
        • If a category doesnt contain any channel, it is hidden from members that dont have the Manage Channels permissions
      • Group List Update
        • It is possible to hide groups from users now, tho i didnt fully implement this feature. This means while you can edit the config.json file to hide certain groups for certain roles, its not possible to do this using the web client/ui. This will be implemented in a feature update.
